# 게임 플로우 스펙 (Game Flow Specification)

> 전투 엔진 위에 "게임"을 만들기 위한 화면 흐름 및 각 화면의 책임 정의

---

## 화면 흐름 개요

```
Loading → Title → Town (Hub) → Formation → Sortie Select → Run/Battle → Result → Town
                    ↑                                                          ↓
                    └──────────────────────────────────────────────────────────┘
```

Town(마을)이 게임의 중심이다. 모든 메타 진행이 여기서 이루어지고, 전투를 나가고 돌아오는 기점이 된다.

---

## §1. Loading (로딩)

### 역할
- 게임 에셋 로드 (스프라이트, 오디오, 폰트)
- 세이브 데이터 로드 (LocalStorage / IndexedDB)
- 첫 진입이면 초기 데이터 생성 (스타터 캐릭터 지급 등)

### 화면 구성
- 로딩 프로그레스 바
- 게임 로고 또는 타이틀 이미지

### 전이
- 로딩 완료 → Title

---

## §2. Title (타이틀)

### 역할
- 게임 진입점
- 세이브 슬롯 선택 (MVP에서는 단일 세이브)

### 화면 구성
- 게임 타이틀 로고
- 메뉴 버튼:
  - **시작하기** (첫 플레이) / **이어하기** (세이브 존재 시)
  - **설정** (볼륨, 해상도 등)

### 전이
- 시작하기/이어하기 → Town
- 설정 → 설정 팝업 (Title 위에 오버레이)

---

## §3. Town — 허브 (마을)

### 역할
게임의 **메타 진행 중심**. HoMM(Heroes of Might and Magic) 도시 화면처럼
건물/시설을 클릭하면 해당 기능 UI가 열리는 구조.

### 컨셉
- 배경: 마을/거점 일러스트 (건물들이 배치된 전경)
- 각 건물은 클릭 가능한 인터랙티브 오브젝트
- 건물 클릭 → 해당 기능 패널/팝업이 열림
- 건물 업그레이드로 기능 확장 가능 (향후)

### 건물 및 기능

#### 3-1. 병영 (Barracks) — 보유 캐릭터
- 보유 캐릭터 목록 표시 (그리드/리스트)
- 캐릭터 상세 정보 확인 (스탯, 장착 액션 카드, 클래스)
- 캐릭터 보유 슬롯 표시 (현재/최대)
- (향후) 슬롯 확장 구매
- (향후) 캐릭터 해고/방출

#### 3-2. 훈련소 (Training Ground) — 캐릭터 강화
- 캐릭터 선택 → 레벨업 (골드 소모)
- 스탯 성장 미리보기
- 레벨별 비용 표시
- 기존 `TrainingSystem.ts`의 `trainCharacter()` 연동

#### 3-3. 작전실 (War Room) — 편성 관리
- → §4. 편성 화면으로 이동
- 프리셋 목록 미리보기

#### 3-4. 출격 게이트 (Sortie Gate) — 전장 선택
- → §5. 출격 선택 화면으로 이동

#### 3-5. 상점 (Shop) — (향후)
- 액션 카드 구매
- 아이템/소모품 구매

#### 3-6. 설정
- 게임 설정 (볼륨, 그래픽 등)
- 세이브/로드

### 화면 구성
- 마을 전경 배경 (2D 일러스트)
- 건물 오브젝트들 (클릭 영역 + 라벨)
- 보유 골드 표시 (상단)
- 영웅 정보 표시 (상단 또는 사이드)

### 전이
- 건물 클릭 → 해당 기능 패널 (오버레이 또는 전체화면)
- 출격 게이트 → Sortie Select
- (향후) 마을 건물 건설/업그레이드 UI

---

## §4. Formation (편성 화면)

### 역할
전투에 내보낼 파티를 구성하는 화면.
지금까지 만든 전투/런 시스템을 **플레이어 선택**으로 바꾸는 관문.

### 화면 구성

#### 4-1. 파티 편성
- **출전 슬롯 4개** (FRONT/BACK 포지션 자유 배치)
- 보유 캐릭터 목록에서 드래그 또는 클릭으로 배치
- 같은 캐릭터 중복 편성 불가

#### 4-2. 영웅 선택
- 영웅 유형 선택 (COMMANDER / MAGE / SUPPORT)
- 영웅 능력 미리보기
- 런 동안 영웅 고정 (변경 불가)

#### 4-3. 액션 카드 편집
- 캐릭터별 액션 슬롯 3개 확인/편집
- 보유 카드 인벤토리에서 장착
- 기본 액션 슬롯으로 리셋 옵션
- (런 진행 중이면 런 카드 인벤토리에서 장착)

#### 4-4. 프리셋
- 현재 편성을 프리셋으로 저장 (이름 지정)
- 저장된 프리셋 불러오기
- 프리셋 삭제
- MVP: 3~5개 프리셋 슬롯

### 전이
- 뒤로 → Town
- 편성 완료 → Sortie Select (또는 Town으로 돌아가서 출격)

---

## §5. Sortie Select (출격 선택)

### 역할
**어떤 전장에 참여할지** 선택하는 화면.
단순히 "런 시작"이 아니라, 다양한 전장 중 하나를 골라 출격.

### 전장(Battlefield) 개념
- 각 전장은 **고유한 테마, 배경, 적 구성, 난이도**를 가짐
- 전장마다 5스테이지 런 구조 (기존 run-system-spec.md)
- 전장별 차별화:
  - **이름/배경**: 다른 지역, 다른 분위기
  - **적 이름/외형**: 전장 테마에 맞는 적 (숲 → 도적단, 화산 → 마족 등)
  - **난이도/스탯 배율**: 전장별 기본 난이도 차이
  - **특수 규칙**: (향후) 전장별 고유 변칙 (독 지형, 화염 장판 등)
  - **보상 차이**: 높은 난이도 = 더 좋은 보상

### 화면 구성
- 전장 목록 (카드형 또는 리스트)
- 각 전장 정보:
  - 이름, 배경 썸네일
  - 난이도 표시 (별점 또는 레벨 범위)
  - 주요 적 타입 미리보기
  - 클리어 상태 (첫 클리어 여부, 최고 기록 등)
  - (향후) 스토리 진행도
- 잠금된 전장은 조건 표시 (이전 전장 클리어 필요 등)

### 전장 예시 (초안)

| 전장 | 테마 | 난이도 | 특징 |
|------|------|--------|------|
| 변방 초원 | 초원/평야 | ★☆☆ | 입문용, 기본 적 구성 |
| 어둠의 숲 | 숲/야간 | ★★☆ | Ranger 비중 높음, 후열 위협 |
| 폐허 요새 | 폐허/성채 | ★★★ | Guard+Disruptor 조합, 포지션 중요 |
| (향후 확장) | ... | ... | ... |

### 전이
- 뒤로 → Town
- 전장 선택 + 출격 확인 → Run 진입 (§6)

## §6. Run — 전투 루프

### 역할
기존 `run-system-spec.md`의 5스테이지 런 구조를 실제 플레이로 실행.

### 런 진입 시
1. 파티 스냅샷 (`preBattleActionSlots` 저장)
2. 영웅 고정
3. 런 카드 인벤토리 초기화 (빈 상태)
4. 리트라이 토큰 1개 지급
5. 선택된 전장의 적 편성 데이터 로드

### 스테이지 루프 (1~5)

```
[스테이지 N 시작]
  ↓
편성 확인/수정 (런 중 편성 조정 가능)
  ↓
전투 (§6-1)
  ↓
결과 (§6-2)
  ↓
  ├─ 승리 → 보상 (§6-3) → 다음 스테이지
  └─ 패배 → 리트라이 OR 런 종료
```

#### §6-1. 전투 화면 (Battle)
- 기존 BattleEngine 시각화
- 유닛 배치 (FRONT/BACK, 좌=플레이어 / 우=적)
- 턴 진행 애니메이션
- 영웅 개입 버튼 (능력 선택 → 타겟 선택 → 큐잉)
- 전투 로그 (이벤트 기반)
- 배속 조절 (1x / 2x / 스킵)

#### §6-2. 전투 결과
- 승패 표시
- 생존 유닛 상태
- 획득 골드
- 패배 시: 리트라이 선택지

#### §6-3. 보상 (승리 시)
- 골드 획득 확정 표시
- **카드 선택**: 5장 제시 → 1장 선택 (기존 `generateCardRewards()`)
- **게스트 멤버**: 스테이지 2~4에서 확률 등장, 수락/거절
- 다음 스테이지 진입 버튼

### 런 종료
- **승리** (5스테이지 클리어): 완주 보너스 + Town 복귀
- **패배** (리트라이 소진): 중도 종료 + Town 복귀
- 공통: 파티 액션 슬롯 원복, 런 카드 인벤토리 폐기, 골드는 유지

### 전이
- 런 종료 → Run Result (§7)

---

## §7. Run Result (런 결과)

### 역할
런 전체의 결과를 요약하고 Town으로 복귀.

### 화면 구성
- 클리어/실패 표시
- 진행한 스테이지 수
- 총 획득 골드
- 획득한 게스트 멤버 (있다면)
- (향후) 업적/기록 갱신 표시

### 전이
- 확인 → Town

## 세이브 데이터 구조 (개요)

Town을 중심으로 영속적으로 저장되는 데이터:

```typescript
interface SaveData {
  // 플레이어 자원
  gold: number;

  // 캐릭터
  characters: CharacterDef[];      // 보유 캐릭터 목록
  maxCharacterSlots: number;       // 보유 슬롯 상한

  // 편성 프리셋
  presets: FormationPreset[];      // 저장된 편성 목록

  // 영웅
  heroType: HeroType;              // 현재 선택된 영웅

  // 전장 진행
  battlefieldProgress: {
    [battlefieldId: string]: {
      unlocked: boolean;
      cleared: boolean;
      bestRecord?: number;         // 최고 스테이지
    };
  };

  // (향후) 마을 건물 레벨
  // (향후) 업적
  // (향후) 설정
}

interface FormationPreset {
  name: string;
  slots: {
    characterId: string;
    position: Position;
  }[];
  heroType: HeroType;
}
```

## MVP 우선순위

### Phase 1 — 최소 플레이 가능
1. Loading (간소화)
2. Title (시작 버튼만)
3. Town (건물 2~3개: 병영, 작전실, 출격 게이트)
4. Formation (기본 편성만, 프리셋은 나중)
5. Sortie Select (전장 1개만)
6. Battle (기존 엔진 시각화)
7. Result (간단한 승패 + 골드)

### Phase 2 — 메타 루프 완성
- 훈련소 (캐릭터 강화)
- 카드 선택 보상 UI
- 전장 2~3개 추가
- 프리셋 기능
- 세이브/로드

### Phase 3 — 확장
- 상점
- 마을 건물 업그레이드
- 특수 전장/이벤트
- 스토리 연출
- 캐릭터 슬롯 확장

## 기술 매핑

| 화면 | Phaser Scene | 비고 |
|------|-------------|------|
| Loading | `BootScene` (기존) | 에셋 로드 확장 |
| Title | `MainMenuScene` (기존) | 리뉴얼 |
| Town | `TownScene` (신규) | 핵심 허브 |
| 병영 | `TownScene` 내 패널 | 오버레이 UI |
| 훈련소 | `TownScene` 내 패널 | 오버레이 UI |
| 편성 | `FormationScene` (신규) | 별도 Scene |
| 출격 선택 | `SortieScene` (신규) | 별도 Scene |
| 전투 | `BattleScene` (기존) | 대폭 확장 |
| 전투 결과 | `BattleScene` 내 패널 | 오버레이 |
| 보상 | `RewardScene` (신규) | 카드 선택 등 |
| 런 결과 | `RunResultScene` (신규) | 별도 Scene |
