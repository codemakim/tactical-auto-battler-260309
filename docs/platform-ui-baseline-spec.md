# 플랫폼 UI 기준 스펙

> Source of Truth — 2026-04-21

## 목적

현재 게임의 1차 타겟 플랫폼과 UI 안정성 기준을 명확히 고정한다.
새 기능 개발 전에 화면이 한쪽으로 치우치거나, 팝업이 의도치 않게 닫히거나,
카드 태그가 영역 밖으로 흐르는 문제를 기본 품질선으로 막는다.

## 타겟 플랫폼

- 1차 타겟은 데스크톱 브라우저다.
- 게임 화면은 16:9 고정 캔버스 `1280 x 720`을 기준으로 설계한다.
- Phaser Scale은 `FIT + CENTER_BOTH`를 유지한다.
- 최소 표시 기준은 `640 x 360`, 최대 표시 기준은 `1920 x 1080`이다.
- 모바일 전용 반응형 UI는 현재 범위가 아니다. 화면 축소 시 같은 16:9 캔버스를 비율 유지로 축소한다.

## 레이아웃 기준

- 브라우저 뷰포트가 16:9보다 넓거나 좁아도 게임 캔버스는 수평/수직 중앙에 있어야 한다.
- DOM 루트와 Phaser Scale Manager가 서로 다른 정렬 기준을 갖지 않도록 한다.
- UI 좌표는 기본적으로 `GAME_WIDTH / GAME_HEIGHT` 기준으로 배치한다.

## 오버레이 기준

- 딤 배경의 패널 바깥 영역을 클릭하면 오버레이를 닫을 수 있다.
- 패널 내부 빈 영역 클릭은 닫기 동작으로 처리하지 않는다.
- 명시적인 닫기 버튼, 취소 버튼, 바깥 딤 클릭만 닫기 의도를 가진다.
- 병영, 훈련소, 상점, 편성 오버레이는 같은 동작 기준을 따른다.

## 액션 미니카드 기준

- 태그는 실제 텍스트 폭과 패딩을 기준으로 배치한다.
- 고정 높이 카드에서 태그 줄 수는 카드 높이 안에 들어가는 만큼만 허용한다.
- 표시 가능한 줄을 넘는 태그는 마지막 줄의 `+N` 배지로 축약한다.
- 태그가 많은 카드도 카드 박스, 가격, 버튼, 다음 슬롯 영역을 침범하지 않아야 한다.

## 비목표

- 전체 UI 리디자인
- 모바일 전용 레이아웃
- 모든 씬의 미술 방향 재정의
- 카드 상세 정보를 늘리는 설명형 UI
