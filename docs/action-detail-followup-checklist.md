# 액션 상세 표시 후속 체크리스트

> Source of Truth — 2026-04-02
>
> 관련 스펙: `docs/action-detail-display-spec.md`
> 플랫폼 기준: `docs/platform-ui-baseline-spec.md`

## 목적

액션 미니 카드/상세 팝업 1차 구현 후 실제 플레이에서 드러난
가독성, 정렬, 팝업 동작 문제를 순차적으로 정리하기 위한 후속 작업 목록이다.

## 우선순위

1. 전용 모노 아이콘 세트 도입
2. 미니 카드 / 상세 팝업 공통 아이콘 적용
3. 상점 미니 카드 태그 오버플로우 대응
4. 오버레이 내부 빈 영역 클릭 시 닫힘 방지
5. 게임 화면 중앙 정렬 점검 및 수정

## 작업 항목

### 1. 전용 모노 아이콘 세트 도입

- [x] 이모지 기반 효과 아이콘 제거 (Codex, 2026-04-02)
- [x] `DAMAGE / SHIELD / HEAL / MOVE / PUSH / BUFF / DEBUFF / DELAY / ADVANCE / SWAP`용 모노 아이콘 규칙 정의 (Codex, 2026-04-02)
- [x] 텍스트 폰트/두께와 어울리는 크기 규칙 정의 (Codex, 2026-04-02)
- [x] 아이콘 교체 후 `actionCardBadges.ts`, `UIActionMiniCard.ts` 출력 일관성 검증 (Codex, 2026-04-02)

핵심 원칙
- 색과 형태는 유지하되 이모지 폭/베이스라인 흔들림을 제거한다.
- 아이콘은 텍스트를 대체하는 보조 정보여야 하며, 의미 전달은 문장형 태그가 우선한다.

### 2. 미니 카드 / 상세 팝업 공통 적용

- [x] 미니 카드와 호버 상세가 같은 아이콘 세트를 사용하도록 맞춘다 (Codex, 2026-04-02)
- [x] 같은 효과는 미니 카드와 상세 팝업에서 같은 이름/아이콘/색 규칙을 유지한다 (Codex, 2026-04-02)
- [x] 상세 팝업도 `상황 / 대상 / 효과` 섹션 분리 없이 플랫 태그 문법 유지 (Codex, 2026-04-02)

핵심 원칙
- “미니 카드에서 본 규칙”과 “상세 팝업에서 본 규칙”이 달라지지 않아야 한다.
- 아이콘 교체 작업은 이 단계까지 포함돼야 완료로 본다.

### 3. 상점 미니 카드 태그 오버플로우 대응

- [x] 태그가 많은 카드에서 줄바꿈 시 카드 박스를 벗어나지 않도록 처리 (Codex, 2026-04-21)
- [x] 상점 카드 높이/슬롯 간격/미니 카드 높이 재조정 (Codex, 2026-04-21)
- [x] 필요 시 상점용 미니 카드의 태그 밀도 규칙 따로 정의 (Codex, 2026-04-21)

검토 포인트
- `Shield Wall`, `Advance Guard`, `Charge` 같은 복합 태그 카드
- 3줄 이상 태그가 생기는 경우
- 가격/영입 버튼과의 충돌 여부
- 고정 높이 미니카드는 표시 가능한 줄 이후를 `+N` 배지로 축약한다.

### 4. 오버레이 내부 빈 영역 클릭 시 닫힘 방지

- [x] 오버레이 dim 바깥 클릭만 닫힘으로 인정 (Codex, 2026-04-21)
- [x] 패널 내부의 비어 있는 영역 클릭은 무시 (Codex, 2026-04-21)
- [x] 병영 / 훈련소 / 상점 공통 동작 확인 (Codex, 2026-04-21)

### 5. 게임 화면 중앙 정렬 점검 및 수정

- [x] 와이드 모니터에서 게임 루트 컨테이너가 우측으로 치우치는 원인 확인 (Codex, 2026-04-21)
- [x] 캔버스/부모 컨테이너/CSS 정렬 기준 점검 (Codex, 2026-04-21)
- [x] 전체 게임 화면이 수평 중앙에 오도록 수정 (Codex, 2026-04-21)

검토 범위
- 브라우저 캔버스 정렬
- Phaser scale manager 설정
- 상위 DOM 레이아웃

## 진행 순서

1. 아이콘 세트 설계와 스펙 반영
2. 미니 카드 / 상세 팝업 공통 적용
3. 상점 오버플로우 수정
4. 오버레이 클릭 동작 수정
5. 게임 화면 중앙 정렬 수정

## 완료 기준

- 이모지 아이콘이 남아 있지 않다
- 미니 카드와 상세 팝업의 표현 규칙이 일치한다
- 상점 카드에서 태그가 박스를 벗어나지 않는다
- 패널 내부 빈 영역 클릭으로 팝업이 닫히지 않는다
- 와이드 화면에서도 게임이 수평 중앙 정렬된다
