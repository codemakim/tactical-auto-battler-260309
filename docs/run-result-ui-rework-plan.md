# RunResult UI Rework Plan

> Deferred plan.
> 지금은 기능 우선으로 두고, 실제 플레이 후 개선 필요성이 확인되면 이 문서를 기준으로 손본다.

---

## 목적

현재 `RunResultScene`은 기능적으로는 충분하지만, 화면 표현은 아직 임시 요약 패널에 가깝다.

이 리워크의 목적은:
- 런 종료 순간을 더 게임답게 마무리하기
- 승리/패배 감정을 더 분명하게 전달하기
- Town 복귀 전 최종 브리핑 화면처럼 보이게 하기

중요:
- 이 작업은 현재 필수 기능이 아니다
- 런 루프와 저장/복귀 흐름이 충분히 안정된 뒤 진행한다

---

## 현재 상태

현재 `RunResultScene`은 아래만 제공한다.
- 승리/패배 타이틀
- 클리어 스테이지
- 획득 골드
- Town 복귀 버튼

이 상태는 MVP로는 충분하지만, 아래 한계가 있다.
- 작전 종료의 분위기가 약함
- 숫자 위계가 약함
- 승리와 패배의 시각 차이가 작음
- 결과 화면만의 아이덴티티가 부족함

---

## 리워크 목표

### 1. 헤더 강화
- 상태 라벨: `RUN COMPLETE` / `RUN FAILED`
- 메인 타이틀:
  - 승리: `OPERATION COMPLETE`
  - 패배: `SQUAD LOST`
- 서브카피:
  - 승리: 완주 보고 느낌
  - 패배: 철수/실패 보고 느낌

### 2. 결과 블록화
- 텍스트 2~3줄 대신 브리핑 카드/요약 패널 형태
- 최소 유지 정보:
  - `STAGES`
  - `GOLD`
- 숫자가 먼저 읽히고, 설명은 보조 정보로 내려감

### 3. 승패 시각 차등
- 승리: 금색/청색 계열
- 패배: 적색/황동색 계열
- 레이아웃은 유지하되 분위기만 달라지게

### 4. Town 복귀 액션 정리
- 기본 버튼은 `RETURN TO TOWN` 하나 유지
- 버튼은 화면 하단 고정
- 추가 액션은 당분간 넣지 않음

---

## 제외 범위

아래는 이 리워크에서 하지 않는다.
- 전장별 통계
- 영웅별 통계
- 전투 로그 다시보기
- 업적/기록 비교
- 게스트 멤버 결과 요약
- 런 중 획득 카드 수 표시

특히 `런 중 카드`는 임시 자원이므로 결과 보상처럼 보이면 안 된다.

---

## 선행 조건

이 작업은 아래가 먼저 안정된 뒤 진행한다.

1. 전체 런 루프가 플레이 가능할 것
- 출격
- 런 진행
- 보상
- 런 종료
- Town 복귀

2. 세이브/로드 흐름이 충분히 안정적일 것

3. 플레이 테스트 후 실제로 결과 화면 개선 필요성이 확인될 것

---

## 추천 작업 순서

1. 현재 `RunResultScene` 플레이 감상 정리
- 무엇이 심심한지
- 무엇이 모자란지
- 무엇이 과한지

2. 레이아웃 목업 정리
- 헤더
- 결과 블록
- 하단 버튼

3. 카피 확정
- 승리/패배 문구
- 보조 설명 문구

4. 구현
- Scene 리디자인
- 필요 시 `RunResultPresentation.ts` 같은 순수 표현 헬퍼 추가

5. 검증
- 기존 결과 계산/정산 테스트 유지
- 필요하면 표현용 순수 함수 테스트 추가

---

## 구현 아이디어 메모

- 상단에 넓은 브리핑 바
- 중앙에 2개 결과 카드:
  - `STAGES`
  - `GOLD`
- 승리면 미세한 금색 강조선
- 패배면 붉은 경고선
- Town 버튼은 과하게 크지 않되 확실히 눌러야 할 마지막 액션처럼 배치

---

## 트리거

아래 중 하나가 생기면 이 계획을 실제 작업으로 승격한다.

- 플레이 중 RunResultScene이 너무 밋밋하다고 느껴짐
- 승리/패배 감정 전달이 약하다는 피드백이 나옴
- 전체 런 루프 기능이 일단 안정화됨
