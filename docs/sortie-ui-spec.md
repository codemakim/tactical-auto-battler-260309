# 출격 선택 UI 스펙 (Sortie UI Specification)

> Source of Truth for `SortieScene`.
> `game-flow-spec.md §5`의 상세 화면 규칙을 정의한다.

---

## §1. 역할

SortieScene은 플레이어가 전장을 고르고 런을 시작하는 화면이다.
여기서 전달해야 하는 정보는 "어디로 출격하는가"와 "얼마나 위험한가"까지다.

이 화면은 보상 예고 화면이 아니다.

즉 SortieScene은 아래만 보여준다.
- 전장 이름
- 전장 분위기/테마
- 난이도
- 적 구성 힌트
- 잠금 여부

아래는 보여주지 않는다.
- 예상 골드
- 액션 카드 보상
- 객원 멤버 보상
- 특정 전장 클리어 시 영구 보상

---

## §2. 표시 정보

### §2.1 카드별 필수 정보
- 전장 이름
- 전장 테마
- 난이도 표시
- 적 미리보기
- 짧은 설명
- 잠금 상태 / 출격 가능 상태

### §2.2 표시 금지 정보
- `보상: 골드 + 액션 카드 + 객원 멤버` 같은 문구
- 런 종료 시 사라지는 임시 자원을 영구 보상처럼 읽히게 만드는 표현
- 아직 구현되지 않은 기능(객원 멤버 등)을 암시하는 UI

---

## §3. 카피 가이드

SortieScene의 카피는 "전장 브리핑" 톤으로 유지한다.

허용:
- `위험도`
- `주요 적`
- `출격 가능`
- `잠김`

비허용:
- `보상`
- `획득 예정`
- `게스트 멤버`
- `액션 카드 획득`

---

## §4. Scene 동작

### §4.1 선택
- 잠금 해제된 전장만 선택 가능
- 선택 시 하단 출격 버튼 라벨이 해당 전장 기준으로 갱신

### §4.2 출격
- 선택된 전장 기준으로 `createRunState()` 호출
- 선택한 전장 ID는 `RunState.battlefieldId`에 저장한다
- 전투 진입 시 해당 전장 ID를 기준으로 전장 배경을 선택한다
- 첫 진입은 `RunMapScene`으로 이동

### §4.3 뒤로 가기
- Town으로 복귀

---

## §5. 디자인 원칙

- SortieScene은 전장 선택 화면이지 보상 예고 화면이 아니다
- 전장 정보는 "난이도와 적 성향" 위주로 압축한다
- 런 보상은 승리 후 RewardScene에서만 공개한다
- 임시 런 자원은 SortieScene과 RunResultScene 어디에서도 영구 보상처럼 보이지 않게 한다

---

## §6. 현재 구현과 차이

현재 코드에는 전장 카드 하단에
`보상: 골드 + 액션 카드 + 게스트 멤버`
문구가 있다.

이 문구는 아래 이유로 제거 대상이다.
- 보상 예고를 SortieScene 책임으로 잘못 확장함
- 객원 멤버는 현재 폐기된 기능임
- 액션 카드는 런 종료 시 사라지는 임시 자원이라 영구 보상처럼 읽히면 안 됨

---

## §7. 구현 판단 기준

다음 `SortieScene` 수정 작업은 아래만 하면 된다.

1. 전장 카드에서 보상 문구 제거
2. 필요하면 그 자리를 전장 분위기/적 정보로 대체
3. 객원 멤버 관련 카피 제거
