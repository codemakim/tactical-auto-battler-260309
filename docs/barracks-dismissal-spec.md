# 병영 방출 스펙 (Barracks Dismissal Specification)

> Draft
> 병영에서 보유 캐릭터를 로스터에서 제거하는 MVP 규칙을 정의한다.
> 목표는 로스터 정리 수단을 제공하되, 런 진행 상태나 편성 무결성을 깨지 않도록 하는 것이다.

---

## §1. 목적

병영에 보유 캐릭터가 계속 쌓이면:

- 신규 영입 판단이 어려워지고
- 로스터 관리가 불편해지며
- 플레이어가 원치 않는 멤버를 정리할 수단이 없어진다

따라서 병영에는 **명시적 방출 기능**이 필요하다.

이 기능은 판매나 분해가 아니라, **캐릭터를 로스터에서 완전히 제거**하는 정리 수단이다.

---

## §2. MVP 범위

### §2.1 포함

- 병영 상세 패널에서 캐릭터 1명을 방출할 수 있다
- 방출 버튼 클릭 시 확인 모달이 한 번 더 뜬다
- 확인 시 해당 캐릭터는 로스터에서 제거된다
- 편성, 프리셋, 선택 상태에서 함께 정리된다
- 불가능한 경우는 버튼 비활성 또는 안내 문구로 막는다

### §2.2 제외

- 방출 보상(골드 환급, 재료 환급)
- 되돌리기(undo)
- 여러 명 일괄 방출
- 잠금/즐겨찾기 시스템
- 방출 로그/묘비/도감 기록

---

## §3. 핵심 원칙

### §3.1 방출은 정리 기능이다

방출은 경제 시스템이 아니다.

- 골드 환급 없음
- 아이템 환급 없음
- 카드 환급 없음

즉, 플레이어가 로스터를 정리하기 위해 사용하는 기능이다.

### §3.2 런 무결성을 깨면 안 된다

진행 중 런에서 참조되는 캐릭터를 병영에서 제거하면 상태가 꼬인다.

따라서 **런 진행 중에는 방출 금지**가 기본이다.

### §3.3 최소 전투 인원은 보장해야 한다

스타터 4명 구조에서 방출을 허용하되, 결과적으로 전투에 필요한 최소 인원보다 적어지면
출격/편성이 막히거나 게임이 멈출 수 있다.

MVP에서는 아래 규칙으로 단순화한다.

- 전체 보유 캐릭터 수가 **4명 이하**이면 방출 불가

즉, 최소 전투 인원 4명은 항상 유지한다.

---

## §4. 방출 가능 조건

아래를 모두 만족해야 한다.

1. 현재 `runState`가 없음
2. 전체 보유 캐릭터 수가 5명 이상
3. 대상 캐릭터가 존재함

---

## §5. 방출 불가 조건

### §5.1 진행 중 런

`runState`가 존재하면 방출 불가.

이유:

- 런 파티 스냅샷
- 편성
- 런 카드 장착 상태
- 저장/이어하기

와 충돌할 수 있다.

UI 처리:

- 버튼 비활성
- 안내 문구 예: `런 진행 중에는 방출할 수 없습니다`

### §5.2 최소 보유 인원 도달

보유 캐릭터 수가 4명 이하이면 방출 불가.

UI 처리:

- 버튼 비활성
- 안내 문구 예: `최소 4명의 멤버는 유지해야 합니다`

---

## §6. UI 흐름

### §6.1 진입점

병영 상세 패널의 선택 캐릭터 영역 하단에 `방출` 버튼을 둔다.

### §6.2 1차 액션

플레이어가 `방출` 버튼 클릭

→ 즉시 삭제하지 않고 확인 모달 표시

### §6.3 확인 모달

권장 카피:

- 제목: `멤버 방출`
- 본문: `{캐릭터 이름}을(를) 로스터에서 방출합니다. 이 작업은 되돌릴 수 없습니다.`
- 버튼:
  - `취소`
  - `방출`

### §6.4 완료

확인 시:

1. 캐릭터를 로스터에서 제거
2. 현재 편성에서 제거
3. 해당 캐릭터를 포함한 프리셋에서 제거
4. 병영 선택 상태 초기화 또는 다음 캐릭터 자동 선택
5. 토스트 표시

예:

- `{캐릭터 이름} 방출`

---

## §7. 데이터 반영 규칙

### §7.1 캐릭터 목록

`GameState.characters`에서 제거한다.

### §7.2 현재 편성

`formation.slots`에서 해당 캐릭터를 제거한다.

### §7.3 프리셋

모든 `presets[].formation.slots`에서 해당 캐릭터를 제거한다.

빈 편성이 된 프리셋은 삭제하지 않는다.
그대로 빈 슬롯 상태 프리셋으로 유지한다.

### §7.4 런 상태

MVP에서는 런 진행 중 방출을 금지하므로 별도 수정하지 않는다.

---

## §8. 저장 규칙

방출은 Town 메타 상태 변경이므로 즉시 저장 대상이다.

즉, 방출 후 새로고침해도:

- 캐릭터가 다시 나타나면 안 된다
- 편성/프리셋에서도 제거된 상태여야 한다

---

## §9. 테스트 기준

최소 계약 테스트:

1. 방출 가능 여부 계산
   - 런 없음 + 캐릭터 5명 이상 → 가능
   - 런 진행 중 → 불가
   - 캐릭터 4명 이하 → 불가
2. 캐릭터 방출 시 `characters`에서 제거된다
3. 캐릭터 방출 시 `formation.slots`에서도 제거된다
4. 캐릭터 방출 시 모든 프리셋에서도 제거된다
5. 저장 후 로드해도 제거 상태가 유지된다

---

## §10. 구현 순서

1. 방출 가능 여부/상태 계산 순수 함수 추가
2. `GameState`에 방출 API 추가
3. 관련 테스트 추가
4. 병영 상세 패널에 `방출` 버튼 + 확인 모달 연결
5. 저장/로드 검증
